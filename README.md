This repository contains a structured GRC portfolio built to demonstrate practical, hands-on knowledge of the four frameworks most relevant to the UK and EU regulatory market — ISO 27001:2022, GDPR, NIS2, and DORA.

Each project produces real, employer-facing deliverables: documents and spreadsheets that could be used in a professional GRC engagement. All projects use fictional companies and anonymised data.

---

## Projects

### ✅ Project 1 — Multi-Framework Controls Mapping

Maps 10 core information security control domains across ISO 27001:2022, GDPR, NIS2, and DORA. Identifies overlapping requirements, framework-unique obligations, and provides a practical recommendation for how an Irish organisation can implement once and satisfy all four frameworks simultaneously.

| Deliverable | Description |
|---|---|
| `GRC_Controls_Mapping.xlsx` | 10-domain controls mapping spreadsheet with overlap analysis |
| `Project7_Executive_Summary.docx` | 1-page board-level summary of findings and recommendations |

**Frameworks:** ISO 27001:2022 · GDPR · NIS2 · DORA  
**Key skills demonstrated:** Multi-framework mapping · Overlap analysis · Regulatory interpretation · Executive communication

### ✅ Project 2 — ISO 27001 ISMS Documentation Pack

Full ISMS documentation set for UrbanIQ Technologies Ltd — a 10-person cybersecurity consultancy serving government and public sector clients. Demonstrates end-to-end ISO 27001 implementation capability from asset identification through to audit readiness.

| Deliverable | Document ID | Description |
|---|---|---|
| `UrbanIQ_Asset_Inventory.xlsx` | TBSOL-DOC-001 | 25-asset information asset inventory across 5 asset types with classification, ownership, and disposal |
| `UrbanIQ_Risk_Register.xlsx` | TBSOL-DOC-002 | 20-risk register with likelihood × impact scoring, treatment decisions, and residual risk |
| `UrbanIQ_SoA.xlsx` | TBSOL-DOC-003 | Full Statement of Applicability — all 93 ISO 27001:2022 Annex A controls with inclusion/exclusion justifications |
| `UrbanIQ_Access_Control_Policy.docx` | TBSOL-POL-001 | Access control policy aligned to NIST 800-63b — covers least privilege, JML process, MFA, and remote access |
| `UrbanIQ_Incident_Response_Policy.docx` | TBSOL-POL-002 | IR policy with P1–P4 classification, 7-phase lifecycle, and simultaneous GDPR/NIS2/DORA reporting timelines |
| `UrbanIQ_Data_Classification_Policy.docx` | TBSOL-POL-003 | 4-level classification scheme — Public, Internal, Confidential, Restricted — with handling, storage, and disposal rules |
| `UrbanIQ_Internal_Audit_Checklist.xlsx` | TBSOL-AUD-001 | 50-question internal audit checklist covering ISO 27001:2022 Clauses 4–10 and key Annex A controls |

**Frameworks:** ISO 27001:2022 · GDPR · NIS2 · DORA  
**Key skills demonstrated:** Asset classification · Risk assessment · SoA authoring · Policy writing · Audit preparation

## Frameworks Covered

| Framework | Scope |
|---|---|
| ISO 27001:2022 | Information security management — 93 Annex A controls across 4 themes |
| GDPR | Personal data protection — Articles 5, 24, 25, 28, 32, 33, 34, 35 |
| NIS2 | Network and information security — Article 21 (10 mandatory measures) |
| DORA | Digital operational resilience for financial entities — Articles 6–13, 18–19, 24, 28–30 |
| NIST CSF | Cybersecurity framework — Identify, Protect, Detect, Respond, Recover |

## About This Portfolio

This portfolio was built as part of a structured self-development programme alongside an MSc in Cybersecurity from Dublin Business School (graduated January 2026). The goal was to move beyond theoretical knowledge and produce tangible, professional-quality GRC deliverables that demonstrate real analytical and documentation capability.

Every project is scoped to reflect the current Irish and EU regulatory environment — particularly the convergence of NIS2, DORA, and GDPR obligations that Irish organisations are navigating in 2025 and 2026.
